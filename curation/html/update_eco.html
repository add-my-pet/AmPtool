update_eco

updates eco-codes in existing my_pet_res.html file

Contents

  • Syntax
  • Description
  • Remarks

Syntax

update_eco(varargin)

Description

Updates eco-codes in existing my_pet_res.html file on the server and writes results in entries_web/my_pet/my_pet_res.html, using current get_eco. Updates are only functional if get_eco changed since the creation of the res-files. Only the eco-codes and labels are updated.

Input:

  • varargin: optional cell array with entry names, or character string with entry or node names (default 'Animalia')

Output:

  • 1 html-file is written in ../entries_web

Remarks

Copies model, COMPLETE, MRE, SMSE (in this sequence) from existing my_pet_res.html file. Only updates eco-codes and assumes spacing details in res-file and texts "COMPLETE = ", "MRE = ", "SMSE = ". To finalize eco-updates, the files in entries_web must be copied to the server, using overwrite. Make sure that AmPeco.html on the server has the latest version and that get_eco_type is run since then.
